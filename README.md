# Statistical Decision Tree

An evidence-aware, bilingual (English/Greek) statistical decision-support application for postgraduate students, doctoral candidates, researchers, and educators.

The app converts a structured six-step research-design interview into:

- a primary statistical-method recommendation;
- reasons and a defensible alternative;
- an audit of the analysis originally planned by the user;
- an assumptions checklist;
- practical SPSS, Jamovi, and R guidance;
- an optional personalized explanation generated by GPT-5.6.

> Educational decision support only. Final analytical choices must reflect disciplinary standards, the data, and expert statistical judgment.

## Architecture

The application deliberately separates statistical selection from generative AI:

1. **Deterministic decision engine** (`src/lib/decision.ts`) selects and audits the statistical method from explicit research-design rules.
2. **GPT-5.6 explanation layer** (`src/lib/ai-explanation.ts`) explains the already-selected recommendation, identifies missing information, provides cautions, and suggests a next step.
3. GPT-5.6 is instructed not to replace or contradict the decision engine and returns a strict JSON schema.
4. The OpenAI API key is read only on the server and is never exposed to the browser bundle.

This hybrid design provides reproducible recommendations while using AI where natural-language interpretation is most useful.

## Main features

- Complete English and Greek interface with saved language preference
- Six-step adaptive assessment
- Deterministic statistical recommendations and conflict auditing
- Assumption-aware alternatives
- SPSS and Jamovi menu guidance

- GPT-5.6 bilingual explanation on demand

## Local installation

Requirements: Node.js 20 or newer and npm.

```bash
git clone https://github.com/smaratina/statistical-decision-tree.git
cd statistical-decision-tree
npm install
cp .env.example .env
npm run dev
```

Open the local URL printed by Vite.

### Configure GPT-5.6

Create an OpenAI API key in your OpenAI Platform account and add it to `.env`:

```env
OPENAI_API_KEY=your_real_key_here
```

Never commit `.env` or expose the key through a `VITE_` environment variable. The repository contains only `.env.example`.

The deterministic application works without an API key. The **Explain with GPT-5.6** action requires the server secret.

## Try the sample scenario

1. Open the landing page.
2. Select **View Example** / **Προβολή παραδείγματος**.
3. The sample asks whether self-regulated learning mediates the relationship between achievement goals and social-emotional learning (`N = 94`).
4. The deterministic engine recommends bootstrap mediation analysis and audits the user's planned linear regression.
5. Select **Explain with GPT-5.6** to generate a language-matched explanation.
6. Review the assumptions and switch among SPSS, Jamovi, and R.

## Verification

```bash
npm test
npm run lint
npm run build
```

The automated tests cover the competition demo's mediation path, Greek localization of the decision, an independent-groups comparison, and the safe underspecified-design fallback.

## Deployment

Deploy the full TanStack Start application to a host that supports its server runtime. Configure `OPENAI_API_KEY` as a protected server-side environment variable in the hosting dashboard. Do not place it in repository settings intended for client-side `VITE_*` variables.

The build produces the deployable `.output` directory:

```bash
npm run build
```

## How Codex, Lovable, and GPT-5.6 were used

### Codex

Codex was used as the primary engineering agent to:

- turn the initial product discussion into a project specification and architecture;
- implement and test the first deterministic statistical decision engine;
- initialize the public GitHub repository;
- inspect and merge the exported Lovable codebase without rewriting the original Git history;
- design and implement the secure GPT-5.6 server integration;
- add structured outputs, error handling, tests, build verification, and documentation;
- review statistical wording and correct the mediation R example.

### Lovable

Lovable was used to create and iteratively refine the polished responsive interface, the six-step interview, bilingual content, report layout, PDF printing, and user-facing controls. Its exported code was then brought back into the Codex-managed repository.

### GPT-5.6 inside the application

GPT-5.6 is called through the OpenAI Responses API only when the user requests an AI explanation. It receives the structured answers and deterministic recommendation, then returns:

- a concise explanation;
- missing information;
- statistical cautions;
- one practical next step.

The response is constrained by a strict JSON schema. GPT-5.6 does not own the final statistical-method selection, which makes the core result reproducible and auditable.

## Privacy and limitations

- Users should not enter names, identifiable participant data, confidential datasets, or API keys into the research-question field.
- The AI explanation is generated content and must be reviewed critically.
- The current decision tree supports common canonical designs and directs underspecified or non-standard cases toward consultation.
- The tool does not execute analyses or validate a dataset.
